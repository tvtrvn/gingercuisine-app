## Shipped (what + why, one line each)

Functional / server:
1. **CAS transition guard** — `updateOrder(…, {expectedStatus})` compare-and-set via
   `updateMany`; 409 + current row on conflict; dashboard sends the status the tablet
   saw and resyncs on 409. Kills the two-tablet last-write-wins race. (Deliberate:
   NO legality matrix — "Reopen as new" completed→new is a feature; staleness, not
   legality, was the bug.) Regression tests bite (fail pre-fix).
2. **Corrupt itemsJson resilience** — `parseItems` maps bad blobs to `[]` + error log;
   one bad row can't 500 the whole board/search.
3. **Tracker terminal stop** — `doneRef` prevents visibility toggles from re-arming
   polling on completed/cancelled orders.
4. **Email placeholder footgun** — recipients now `string | undefined`; senders
   warn+skip when unset (no more silent example.com mail); contact page hides the
   email link when unset.
5. **Simplifications** — rateLimit `isConfigured` dedupe; search/pause routes use
   `dashboardRateLimitKey`; `statusRank` derived from `ORDER_STATUS_ORDER`.
6. **Phone backspace fix** — delete-last-digit fallback only when caret at end;
   mid-string formatting deletes no longer eat a digit (5 new unit tests, mutation-checked).

Storefront UX (customer-facing):
7. **Cart persists across reloads** — sessionStorage `gc_cart_v1`, sanitized hydrate
   (shape + finite quantity/unitPrice + 1–99 clamp), per-tab only (no zombie carts).
8. **Menu progressive disclosure** — collapsed cards w/ one-tap default Add +
   per-card "Customize"; collapsed list sheds hundreds of DOM controls; one-tap
   parity with old behavior validated.
9. **Mobile filter compaction** — search + "Filters" toggle at <sm; dishes in first viewport.
10. **768px grid fix** — single column until `lg` (was 2-col squeeze, one-word-per-line names).
11. **Floating-element dock** — sticky bar hidden on `/order*`; FAB/toast/popover lift
    when bar visible; scroll-top hidden on mobile while bar up; main bottom padding
    raised (safe-area aware); "Cart 0" badge hidden at zero.
12. **Checkout sheet polish** — −/+ qty stepper (44px, capped 99), notes behind
    "Add note", single prices-notice, microcopy "Split one off for its own note",
    sheet slide-in animation, useMediaQuery lazy init (no hydration flash).
13. **Confirmation recovery** — bad/expired link shows a recovery card + "Back to
    ordering" instead of a false success header.

Docs: walkthrough.md updated (§7.7 store, §7.13 board, §7.14 cart, /menu description,
glossary, troubleshooting). e2e/order-pricing.spec.ts minimally updated (opens
Customize first); no assertions weakened.
